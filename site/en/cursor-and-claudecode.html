Cursor and ClaudeCode

Use Cursor IDE and Claude Code CLI from your phone or any device.

Chat with Cursor and Claude Code through the Companion app. Open projects, run AI agents, execute commands, and see results — all from your mobile device or any channel.

Overview

HomeClaw integrates with Cursor IDE and Claude Code CLI through a bridge server running on your dev machine. This allows you to:

  • Open projects in Cursor IDE remotely
  • Run AI agents to fix bugs, write tests, refactor code
  • Execute shell commands and see results
  • Continue sessions across multiple messages

How It Works

Layer Runs on Role
Companion app Phone/Tablet UI, sends requests to Core
HomeClaw Core Server/LAN Routes to Cursor/ClaudeCode plugins
Cursor Bridge Dev machine Runs Cursor/Claude CLI commands
Cursor/Claude CLI Dev machine AI agent execution
Note: All file paths refer to the bridge machine's filesystem, not your phone. The bridge runs where Cursor and Claude Code are installed.

Requirements

Before setting up Cursor and ClaudeCode friends, ensure you have the necessary tools installed on your dev machine.

For Cursor

  • Cursor IDE — Download from cursor.com
  • Cursor CLI — Install shell command (see below)
  • Cursor Agent — CLI for running AI tasks

For Claude Code

  • Claude Code CLI — Install from npm
  • Anthropic API key — For Claude Code authentication

Bridge Server

The bridge server is included with HomeClaw and runs on your dev machine to connect Core to Cursor/Claude CLI.

Install Cursor CLI and Claude Code

Option 1: Install with HomeClaw Script

The easiest way — install both during HomeClaw setup:

Mac / Linux

Install with Cursor CLI and Claude Code

HOMECLAW_INSTALL_CURSOR_CLI=1 HOMECLAW_INSTALL_CLAUDE_CODE=1 bash install.sh

Windows (PowerShell)

Install with Cursor CLI and Claude Code

$env:HOMECLAW_INSTALL_CURSOR_CLI="1"; $env:HOMECLAW_INSTALL_CLAUDE_CODE="1"; .\install.ps1

Option 2: Manual Installation

Cursor CLI (Mac / Linux)

Install Cursor CLI

curl https://cursor.com/install -fsS | bash

Cursor CLI (Windows PowerShell)

Install Cursor CLI on Windows

irm 'https://cursor.com/install?win32=true' | iex

Cursor Shell Command

Install the cursor shell command to open projects from terminal:

  1. Open Cursor IDE
  2. Press Cmd+Shift+P (Mac) or Ctrl+Shift+P (Windows/Linux)
  3. Type "Shell Command: Install 'cursor' command"
  4. Restart your terminal

Claude Code CLI

Install Claude Code CLI

npm install -g @anthropic-ai/claude-code

Authenticate Claude Code

Login to Claude Code

claude login

Authenticate Cursor Agent

Login to Cursor Agent

agent login

Or set environment variable:

Set Cursor API key

export CURSOR_API_KEY="your-cursor-api-key"

Configuration

Configure the Cursor Bridge in config/skills_and_plugins.yml.

Auto-start Bridge (Recommended)

Have Core automatically start the bridge server:

Enable auto-start in skills_and_plugins.yml

cursor_bridge_auto_start: true
cursor_bridge_port: 3104

Session Continuation

Enable session continuation so the AI remembers context across messages:

Enable session continuation

cursor_bridge_cursor_continue_session: true
cursor_bridge_claude_continue_session: true

Specify CLI Paths (If Needed)

If the bridge can't find agent or claude, specify full paths:

Specify CLI paths (Windows example)

cursor_bridge_agent_path: "C:\\Users\\You\\AppData\\Local\\Programs\\cursor\\agent.exe"
claude_code_path: "C:\\Users\\You\\AppData\\Local\\Programs\\claude\\claude.cmd"

API Keys

Set API keys if authentication fails:

Set API keys

cursor_bridge_cursor_api_key: "your-cursor-api-key"

Full Configuration Example

Complete bridge configuration

cursor_bridge_auto_start: true
cursor_bridge_port: 3104
cursor_bridge_forward_logs: true
cursor_bridge_cursor_continue_session: true
cursor_bridge_claude_continue_session: true
cursor_bridge_cursor_cli_path: "/usr/local/bin/cursor"
cursor_bridge_agent_path: "/usr/local/bin/agent"
claude_code_path: "/usr/local/bin/claude"

Add Cursor/ClaudeCode Friend

Add the Cursor or ClaudeCode friend to your user configuration.

Using Portal (Recommended)

  1. Start Portal: python -m main portal
  2. Open http://127.0.0.1:18472
  3. Navigate to User Management
  4. Select your user
  5. Click "Add Friend"
  6. Select preset: Cursor or ClaudeCode
  7. Save changes

Editing user.yml

Add friends manually to config/user.yml:

Add Cursor friend

users:
  - name: Alice
    id: alice
    username: alice
    password: your-password
    friends:
      - name: HomeClaw
      - name: Cursor
        preset: cursor

Add ClaudeCode friend

users:
  - name: Alice
    id: alice
    username: alice
    password: your-password
    friends:
      - name: HomeClaw
      - name: ClaudeCode
        preset: claudecode

Restart Core after making changes:

Restart Core

python -m main start

Use from Companion App

Chat with Cursor and ClaudeCode friends from your phone or tablet.

Open the Companion App

  1. Open Companion app on your device
  2. Connect to your HomeClaw Core
  3. Tap on Cursor or ClaudeCode in your friends list

Example Commands for Cursor

  • "Open /Users/alice/myproject in Cursor"
  • "Run Cursor agent to add unit tests for auth module"
  • "Run npm test and show me the results"
  • "What's the status of the current project?"
  • "Fix the bug in the login component"

Example Commands for ClaudeCode

  • "Set project to /Users/alice/myapp"
  • "Run Claude to refactor the database module"
  • "Run Claude agent to fix the failing tests"
  • "Run git status in the project"
  • "Add error handling to the API endpoints"

Session Controls

The Companion app provides quick toggles:

  • Cursor: cursor_agent_yolo toggle — enables --yolo flag for auto-running without prompts
  • ClaudeCode: claude_skip_permissions toggle — enables --dangerously-skip-permissions

Clear Session

To start a fresh session:

  • "Clear cursor session"
  • "New cursor session"
  • "Clear claude session"

Claude Code MCP Server

Besides the bridge (which runs Claude Code CLI), you can use Claude Code as an MCP server — exposing its built-in tools directly to HomeClaw's LLM.

What You Get

  • HomeClaw's LLM can call Claude Code tools by name: Read, Write, Edit, Bash, Task, WebFetch, etc.
  • Fine-grained control from any friend that has MCP tools enabled

Setup MCP Server

First, install the MCP client:

Install MCP client

pip install mcp

Auto-Register (Recommended)

In config/skills_and_plugins.yml:

Auto-register Claude Code MCP server

tools:
  mcp:
    enabled: true
    auto_register_claude_code: true

Manual Configuration

Manual MCP server configuration

tools:
  mcp:
    enabled: true
    servers:
      claude-code:
        transport: stdio
        command: claude
        args: [mcp, serve]
        env: {}

Bridge vs MCP

Method Best For
Bridge (ClaudeCode friend) Natural-language tasks: "fix the bug", "add tests". Claude Code CLI runs autonomously.
MCP server Fine-grained tool calls: HomeClaw's LLM can call Read, Write, Edit, Bash by name.

You can use both: ClaudeCode friend for interactive CLI tasks, and MCP for programmatic tool access.

Tips

Bridge Must Be Running

The bridge server must be running on your dev machine for Cursor/ClaudeCode friends to work. With cursor_bridge_auto_start: true, Core starts it automatically.

Check Bridge Status

Check bridge health

curl http://127.0.0.1:3104/health

Remote Access

If Core runs on a different machine than the bridge:

  1. Set base_url in plugin config to bridge machine's IP
  2. Allow inbound TCP 3104 on bridge machine's firewall
  3. Use Tailscale or VPN for secure access

Session Continuation

With session continuation enabled, the AI remembers context across messages. Each project has its own session, so switching projects maintains separate contexts.

View Logs

Enable log forwarding to see bridge output in Core's terminal:

Enable bridge logs

cursor_bridge_forward_logs: true

Security

Warning: The bridge runs on your dev machine and can execute arbitrary shell commands. Only run it in a trusted environment. If exposing the bridge port, consider adding API key authentication.
