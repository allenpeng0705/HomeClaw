Cursor and Claude Code

Remote coding assistance from your Companion app.

Use Cursor and Claude Code from the HomeClaw Companion app to code on your projects remotely. Send tasks, review results, and manage your codebase from anywhere.

Overview

HomeClaw integrates with Cursor and Claude Code through a bridge server running on your development machine. This allows you to:

  • Open projects in Cursor or set working directory for Claude Code
  • Run AI agents to perform coding tasks remotely
  • Execute commands and see the results
  • Continue sessions across multiple messages

Architecture

Component Runs on Role
Companion app Phone / Desktop UI, sends requests to Core
HomeClaw Core Server / Home PC Routes requests to bridge
Cursor Bridge Dev machine Runs Cursor/Claude CLI
Cursor CLI / Claude Code Dev machine AI coding agents
Important: All file paths in your messages refer to the bridge machine's filesystem, not your phone. The phone does not mount the repository.

Requirements

For Cursor

  • Cursor IDE installed on your dev machine
  • Cursor CLI (comes with Cursor IDE)
  • Cursor API key (from Cursor account)
  • Python 3.11+ for running the bridge

For Claude Code

  • Claude Code CLI installed
  • Anthropic API key OR Minimax API key (or other Anthropic-compatible provider)
  • Python 3.11+ for running the bridge

For Mobile Access

  • Tunnel service (Pinggy, Cloudflare Tunnel, Ngrok, or Tailscale)
  • API key authentication enabled in HomeClaw Core

Install Cursor CLI

Cursor CLI comes bundled with Cursor IDE. After installing Cursor, the CLI should be available.

Verify Installation

Check Cursor CLI

cursor --version

If CLI is Not in PATH

Add Cursor to your PATH or use the full path:

macOS

macOS Cursor path

/Applications/Cursor.app/Contents/Resources/app/bin/cursor

Windows

Windows Cursor path

%LOCALAPPDATA%\Programs\cursor\resources\app\bin\cursor.cmd

Linux

Linux Cursor path

/usr/share/cursor/resources/app/bin/cursor

Cursor CLI Commands

  • cursor agent "task" - Run Cursor agent with a task
  • cursor --help - Show help
  • cursor . - Open current directory in Cursor

Install Claude Code CLI

Install Claude Code CLI using npm:

Install via npm

Install Claude Code

npm install -g @anthropic-ai/claude-code

Verify Installation

Check Claude Code

claude --version

Claude Code CLI Commands

  • claude "task" - Run Claude with a task
  • claude -p "task" - Print mode (non-interactive)
  • claude --continue - Continue last session
  • claude --resume <session_id> - Resume specific session
  • claude mcp serve - Run as MCP server

Configuration Location

Claude Code stores configuration in:

  • macOS/Linux: ~/.claude/settings.json
  • Windows: C:\Users\<you>\.claude\settings.json

Cursor API Key

Cursor requires an API key for using the agent CLI.

Get Your API Key

  1. Open Cursor IDE
  2. Sign in to your Cursor account
  3. Go to Settings → Account
  4. Find or generate your API key

Set API Key

Set the Cursor API key as an environment variable:

macOS/Linux

Set Cursor API key (macOS/Linux)

export CURSOR_API_KEY="your-cursor-api-key"

Windows (PowerShell)

Set Cursor API key (Windows)

$env:CURSOR_API_KEY="your-cursor-api-key"

Windows (Command Prompt)

Set Cursor API key (Windows CMD)

set CURSOR_API_KEY=your-cursor-api-key

Persistent Configuration

Add to your shell profile for persistence:

Add to ~/.bashrc or ~/.zshrc

echo 'export CURSOR_API_KEY="your-cursor-api-key"' >> ~/.bashrc
source ~/.bashrc

Claude Code Configuration

Configure Claude Code by editing ~/.claude/settings.json:

Basic Configuration

~/.claude/settings.json

{
  "env": {
    "ANTHROPIC_API_KEY": "your-anthropic-api-key"
  }
}

Configuration Options

Option Description
ANTHROPIC_API_KEY Anthropic API key (official)
ANTHROPIC_AUTH_TOKEN Bearer token for auth
ANTHROPIC_BASE_URL Custom API endpoint

Using with Custom Endpoint

Custom endpoint configuration

{
  "env": {
    "ANTHROPIC_BASE_URL": "https://your-gateway.com/anthropic",
    "ANTHROPIC_API_KEY": "your-key"
  }
}

Using Minimax with Claude Code

You can use Minimax (or other Anthropic-compatible providers) with Claude Code instead of the official Anthropic API.

Why Use Minimax?

  • Cost-effective alternative to Anthropic
  • Compatible with Anthropic API format
  • Available in regions where Anthropic may be restricted

Minimax Configuration

Minimax requires only Bearer auth (not X-Api-Key). The bridge automatically handles this:

~/.claude/settings.json for Minimax

{
  "env": {
    "ANTHROPIC_BASE_URL": "https://api.minimax.io/anthropic",
    "ANTHROPIC_AUTH_TOKEN": "your-minimax-api-key"
  }
}
Note: When the base URL contains "minimax", the bridge automatically clears ANTHROPIC_API_KEY to avoid auth conflicts. Only ANTHROPIC_AUTH_TOKEN is sent.

Get Minimax API Key

  1. Sign up at minimax.io
  2. Go to API settings
  3. Generate an API key
  4. Add to your Claude Code configuration

Switching Between Anthropic and Minimax

You have two options:

Option A: Edit settings.json

Manually edit ~/.claude/settings.json when switching:

  • For Anthropic: Set ANTHROPIC_API_KEY and remove or change ANTHROPIC_BASE_URL
  • For Minimax: Set ANTHROPIC_BASE_URL to Minimax endpoint and use ANTHROPIC_AUTH_TOKEN

Option B: Use Multiple Config Files

Create separate config files and point HomeClaw to the right one:

In config/skills_and_plugins.yml

cursor_bridge_claude_settings_path: "C:\\Users\\\\.claude\\settings.minimax.json"

Then restart Core. To switch back to Anthropic, change the path to settings.json.

CC Switch Tool

CC Switch is a helper tool for quickly switching between Claude Code configurations.

What CC Switch Does

  • Manage multiple Claude Code configuration profiles
  • Switch between Anthropic, Minimax, and other providers
  • Quickly enable/disable different API endpoints

Installation

Install CC Switch

npm install -g cc-switch

Usage

CC Switch commands

cc-switch list
cc-switch use anthropic
cc-switch use minimax
cc-switch current

Configuration Profiles

Create profiles in ~/.claude/profiles/:

Profile example: ~/.claude/profiles/minimax.json

{
  "env": {
    "ANTHROPIC_BASE_URL": "https://api.minimax.io/anthropic",
    "ANTHROPIC_AUTH_TOKEN": "your-minimax-key"
  }
}

Profile example: ~/.claude/profiles/anthropic.json

{
  "env": {
    "ANTHROPIC_API_KEY": "your-anthropic-key"
  }
}

Integration with HomeClaw

When using CC Switch, the active profile is symlinked to ~/.claude/settings.json. HomeClaw will use whichever profile is currently active.

Cursor Bridge Setup

The Cursor Bridge is a small HTTP server that runs on your dev machine and handles requests from HomeClaw Core.

Start the Bridge

Start Cursor Bridge

python -m external_plugins.cursor_bridge.server

By default, the bridge runs on port 3104.

Environment Variables

Variable Default Description
CURSOR_BRIDGE_PORT 3104 Bridge server port
CURSOR_BRIDGE_CWD . Default working directory
CURSOR_BRIDGE_API_KEY - API key for bridge auth
CURSOR_BRIDGE_CURSOR_CONTINUE 1 Enable Cursor session continuation
CURSOR_BRIDGE_CLAUDE_CONTINUE 1 Enable Claude session continuation

Auto-Start by Core

HomeClaw Core can automatically start the bridge when needed. Configure in config/skills_and_plugins.yml:

Auto-start configuration

cursor_bridge_auto_start: true
cursor_bridge_port: 3104

Security

If the bridge is exposed beyond localhost, set an API key:

Set bridge API key

export CURSOR_BRIDGE_API_KEY="your-secure-key"

Core will send this key in the X-HomeClaw-Bridge-Key header.

Configure in HomeClaw

Configure Cursor and Claude Code in HomeClaw's config/skills_and_plugins.yml.

Session Continuation

Enable session continuation so you can have multi-turn conversations:

Enable session continuation

cursor_bridge_cursor_continue_session: true
cursor_bridge_claude_continue_session: true

Claude Settings Path

Specify a custom Claude settings file:

Custom Claude settings path

cursor_bridge_claude_settings_path: "C:\\Users\\\\.claude\\settings.minimax.json"

Add Friends in user.yml

Add Cursor and ClaudeCode friends for your users:

Add friends in config/user.yml

users:
  - id: yourname
    name: Your Name
    username: yourname
    password: your-password
    friends:
      - name: HomeClaw
      - name: Cursor
        preset: cursor
      - name: ClaudeCode
        preset: claudecode

Plugin Configuration

The plugins are configured in plugins/CursorBridge/plugin.yaml and plugins/ClaudeCodeBridge/plugin.yaml:

Plugin base URL (if Core and bridge on different machines)

config:
  base_url: "http://192.168.1.100:3104"

Mobile Setup

To use Cursor and Claude Code from your phone, you need to set up remote access.

Prerequisites

  • HomeClaw Core running on your server/home PC
  • Cursor Bridge running on your dev machine
  • Tunnel service for remote access

Step 1: Set Up Tunnel

Choose one of the following methods:

Pinggy (Built-in)

Configure Pinggy in core.yml

pinggy:
  token: your-pinggy-token
  enabled: true

Cloudflare Tunnel

Start Cloudflare tunnel

cloudflared tunnel --url http://localhost:9000

Ngrok

Start Ngrok

ngrok http 9000

Tailscale

Install Tailscale on both your phone and server for a private network connection.

Step 2: Enable API Key Authentication

Enable auth in config/core.yml

auth_enable: true
auth_api_key: your-secure-api-key

Step 3: Configure Companion App

  1. Open Companion app
  2. Enter the tunnel URL as Core URL (e.g., https://your-tunnel.trycloudflare.com)
  3. Enter the API key
  4. Test connection

Network Architecture for Mobile

From To Connection
Phone Core (via tunnel) HTTPS
Core Bridge HTTP (local or remote)
Bridge Cursor/Claude CLI Local process
Important: If Core and Bridge are on different machines, ensure Core can reach the Bridge URL. Configure base_url in the plugin YAML accordingly.

Use from Companion App

Once configured, use Cursor and Claude Code from the Companion app.

Select Friend

  1. Open Companion app
  2. Tap on Cursor or ClaudeCode in the friend list
  3. Start chatting

Example Commands

Open Project

  • "Open D:\myproject in Cursor"
  • "Open /Users/me/repos/myapp in Cursor"
  • "Set working directory to /home/user/project"

Run Agent Tasks

  • "Add unit tests for the auth module"
  • "Fix the bug in UserService"
  • "Create a README with installation instructions"
  • "Refactor the database connection code"

Run Commands

  • "Run npm test"
  • "Run pip install -r requirements.txt"
  • "Execute git status"

Session Management

  • "Clear cursor session" - Start fresh session
  • "New claude session" - Start new Claude session
  • "What's the current project?" - Check active directory

Permission Toggles

In the Companion app, you can toggle:

  • Cursor: cursor_agent_yolo - Auto-run mode (skips confirmations)
  • Claude Code: claude_skip_permissions - Skip permission checks

Async Mode

The Companion app automatically uses async mode for Cursor and ClaudeCode friends. This means:

  • Request returns immediately with status 202
  • App polls for results
  • Works well for long-running tasks (minutes)
  • Avoids timeout issues on cellular networks

Progress Preview (Cursor)

For Cursor agent tasks, you may see live progress updates while the agent is running. The bridge streams output so you can see what's happening.

Session Continuation

Both Cursor and Claude Code support session continuation, allowing multi-turn conversations.

How It Works

  1. First message starts a new session
  2. Session ID is saved per project directory
  3. Next message in same project continues the session
  4. Agent remembers context from previous messages

Configuration

Enable in skills_and_plugins.yml

cursor_bridge_cursor_continue_session: true
cursor_bridge_claude_continue_session: true

Session Storage

Sessions are stored in ~/.homeclaw/cursor_bridge_state.json:

Session state structure

{
  "cursor_active_cwd": "/path/to/project",
  "cursor_sessions": {
    "/path/to/project": "session-id-123"
  },
  "claude_active_cwd": "/path/to/project",
  "claude_sessions": {
    "/path/to/project": "session-id-456"
  }
}

Starting Fresh

To start a new session instead of continuing:

  • Say "clear cursor session" or "new cursor session"
  • Say "clear claude session" or "new claude session"
  • Or set cursor_bridge_cursor_continue_session: false in config

Best Practices

  • Short follow-ups: With continuation enabled, send short follow-up messages like "now add tests"
  • Project context: Sessions are tied to project directories
  • Clear when needed: Use "clear session" phrases when you want to start fresh
Tip: Session continuation is especially useful on mobile where you might send multiple short messages. The agent remembers the context from your previous messages in the same project.
