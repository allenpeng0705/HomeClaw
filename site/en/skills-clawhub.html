Skills from ClawHub

Reuse OpenClaw skills from the ClawHub registry.

ClawHub is a registry of OpenClaw skills that you can search, download, convert, and install into HomeClaw. Extend your HomeClaw with community-contributed skills for various tasks.

What is ClawHub

ClawHub is a registry of OpenClaw skills contributed by the community. These skills can be used with HomeClaw after automatic conversion.

Skill Types

  • Instruction-only skills: Provide instructions to the LLM (e.g., API documentation, workflows)
  • Script-based skills: Include Python/JavaScript scripts for specific tasks
  • Integration skills: Connect to external services (Slack, Notion, Outlook, etc.)

How It Works

  1. Search: Find skills in the ClawHub registry
  2. Download: Download the OpenClaw skill to your machine
  3. Convert: Automatically convert to HomeClaw format
  4. Install: Place in external_skills/ folder
  5. Use: Skills are loaded and available in your next chat

Set Up ClawHub Token

Before you can search and install skills, you need to authenticate with ClawHub.

Option 1: Token in Config (Recommended)

Add your ClawHub token directly to config/core.yml:

Configure ClawHub token in core.yml

clawhub_token: "YOUR_TOKEN_FROM_CLAWHUB_AI"

Get your token from clawhub.ai after signing in with GitHub.

Option 2: CLI Login

Use the ClawHub CLI to authenticate:

Login via CLI

clawhub login

This opens a browser for GitHub OAuth. Complete the authorization on the same machine running Core.

Option 3: Token Login (Workaround)

If browser login fails, use token-based login:

Login with token

clawhub login --no-browser --token YOUR_TOKEN

Verify Login

Check login status

clawhub whoami
Note: When using the Companion app or Portal, the token in core.yml is used automatically. You don't need to login separately.

Search Skills

Search for skills in the ClawHub registry using Companion app, Portal, or CLI.

Option 1: Companion App

  1. Open Companion app
  2. Go to Settings → Skills
  3. Use the search bar to find skills
  4. Browse results and view skill details

Option 2: Portal

  1. Start Portal: python -m main portal
  2. Open browser to http://127.0.0.1:18472
  3. Navigate to Skills section
  4. Search and browse available skills

Option 3: CLI

Search skills via CLI

python -m main skills search "slack"
python -m main skills search "email"
python -m main skills search "slides"

Install Skills

Install skills from ClawHub. The skill is downloaded, converted, and placed in your external_skills/ folder.

Option 1: Companion App

  1. Search for a skill
  2. Click Install on the skill card
  3. Wait for download and conversion
  4. Skill is ready to use in your next chat

Option 2: Portal

  1. Search for a skill in Portal
  2. Click Install
  3. The skill is converted and added to external_skills/

Option 3: CLI

Install skill via CLI

python -m main skills install skill-name@1.0.0

What Happens During Install

  1. Download: OpenClaw skill is downloaded to downloads/ folder
  2. Locate: HomeClaw finds the installed skill folder
  3. Convert: OpenClaw format is converted to HomeClaw format (SKILL.md, scripts/, references/)
  4. Install: Converted skill is placed in external_skills/
  5. Load: Skill is loaded on next Core request

Skill Folders

HomeClaw uses multiple skill folders for organization.

Built-in Skills

Location: skills/ folder in your HomeClaw directory

  • Pre-installed skills that come with HomeClaw
  • Protected - cannot be removed via Companion or Portal
  • Examples: html-slides, ppt-generation, image-generation

External Skills

Location: external_skills/ folder (default)

  • Skills installed from ClawHub
  • Can be removed via Companion, Portal, or CLI
  • Configured via external_skills_dir in core.yml

Custom Skill Folders

Add additional skill folders in config/core.yml:

Configure extra skill directories

skills_dir: skills
external_skills_dir: external_skills
skills_extra_dirs:
  - /path/to/custom/skills
  - /another/skills/folder

Loading Order

Skills are loaded in this order (first match wins):

  1. skills_dir (built-in)
  2. external_skills_dir (ClawHub installed)
  3. skills_extra_dirs (custom folders)

Configure via Portal

You can configure skill folders through Portal:

  1. Open Portal at http://127.0.0.1:18472
  2. Go to Settings → Skills
  3. Add or modify skill folder paths
  4. Restart Core for changes to take effect

Maton API Gateway

One important skill from ClawHub is maton-api-gateway-1.0.0, which enables access to many web services.

What is Maton?

Maton provides an API gateway with managed OAuth for 100+ services:

  • Communication: Slack, Discord, Microsoft Teams
  • Email: Gmail, Outlook
  • Productivity: Notion, Airtable, Trello
  • CRM: HubSpot, Salesforce
  • Finance: Stripe
  • And more: Google Workspace, GitHub, Calendly, etc.

How It Works

  • Single API key (MATON_API_KEY) for all services
  • Maton handles OAuth tokens for each connected service
  • Call native APIs through the gateway

Setup

  1. Sign up at maton.ai
  2. Get your API key from settings
  3. Install the skill: python -m main skills install maton-api-gateway@1.0.0
  4. Set the API key in environment or skill config

Set Maton API key

export MATON_API_KEY="your-maton-api-key"

Connect Services

  1. Create a connection for the service you want to use (e.g., Slack)
  2. Open the OAuth URL in a browser
  3. Authorize the connection
  4. Now you can call that service's API through the gateway
Tip: The Maton API Gateway skill is especially useful for integrating with services that require OAuth, as it handles all the token management for you.

Manage Skills

List Installed Skills

List skills via CLI

python -m main skills list

Remove Skills

Remove skills from external_skills/ (built-in skills in skills/ are protected):

Option 1: Companion App

  1. Go to Settings → Skills
  2. Find the skill you want to remove
  3. Click Remove

Option 2: Portal

  1. Open Portal
  2. Navigate to Skills
  3. Click Remove on the skill

Option 3: CLI

Remove skill via CLI

python -m main skills remove skill-name

Disable Skills Temporarily

Add skill names to skills_disabled in config/core.yml:

Disable specific skills

skills_disabled:
  - skill-name-1
  - skill-name-2

Skill Vector Search (RAG)

When you have many skills, enable vector search to find relevant skills automatically:

Enable skill RAG in skills_and_plugins.yml

skills_use_vector_search: true

When enabled, Core creates a vector store of skills and retrieves relevant ones based on your query similarity.
