Skills from ClawHub

Extend HomeClaw with skills from the ClawHub registry.

ClawHub is a public skill registry for OpenClaw-compatible skills. HomeClaw can search, download, convert, and install these skills to extend your assistant's capabilities.

Overview

ClawHub (clawhub.ai) is a public registry for OpenClaw skills. It hosts versioned skill bundles with metadata, tags, and discovery features.

What are Skills?

Skills are reusable workflows that extend your AI assistant's capabilities. Each skill is a folder containing:

  • SKILL.md — Description, instructions, and configuration (YAML frontmatter + Markdown body)
  • scripts/ — Optional Python, Shell, or JavaScript scripts
  • references/ — Optional documentation and examples

Available Skills

Examples of skills available on ClawHub:

  • maton-api-gateway — Connect to 100+ web services (Gmail, Google Calendar, Slack, Notion, Salesforce, HubSpot, etc.) via unified API
  • summarize — Summarize documents and articles
  • weather — Get weather information
  • html-slides — Create HTML presentations
  • ppt-generation — Generate PowerPoint slides
  • image-generation — Generate images using AI
  • linkedin-writer — Write LinkedIn posts
  • And many more...
Recommended: The maton-api-gateway skill is essential for connecting HomeClaw to 100+ popular web services including Gmail, Google Workspace, Slack, Notion, Salesforce, HubSpot, Stripe, and many more. It provides a unified API interface for all these services.

How It Works

  1. Search — Find skills on ClawHub by query
  2. Download — Download OpenClaw skill to staging folder
  3. Convert — Convert to HomeClaw format
  4. Load — Core loads the skill automatically

Setup ClawHub Token

To use ClawHub without browser authentication, set up your token in the config.

Get Your Token

  1. Go to clawhub.ai
  2. Sign in or create an account
  3. Get your API token from account settings

Configure in core.yml

Add your ClawHub token to config/core.yml:

Configure ClawHub token

clawhub_token: "clh_YourTokenHere"

Configure Directories

HomeClaw uses these directories for skills:

Skills directories in core.yml

skills_dir: skills
external_skills_dir: external_skills
clawhub_download_dir: downloads
skills_extra_dirs: []
Directory Purpose Default
skills_dir Built-in skills (protected) skills/
external_skills_dir Converted ClawHub skills external_skills/
clawhub_download_dir Staging for downloads downloads/
skills_extra_dirs Additional skill folders []

Search Skills

Search for skills on ClawHub using the Companion app, Portal, or CLI.

Using Companion App

  1. Open Companion app
  2. Go to Settings → Skills
  3. Click Search Skills
  4. Enter your search query
  5. Browse results and select a skill to install

Using Portal

  1. Start Portal: python -m main portal
  2. Open http://127.0.0.1:18472
  3. Navigate to Skills section
  4. Use the search box to find skills
  5. Click Install on desired skill

Using CLI

Search skills via CLI

python -m main skills search "weather"

Search with limit

python -m main skills search "slides" --limit 10

Install Skills

Install skills from ClawHub to extend your assistant's capabilities.

Installation Flow

  1. Download — Skill is downloaded to downloads/ folder
  2. Convert — OpenClaw format is converted to HomeClaw format
  3. Install — Converted skill is placed in external_skills/
  4. Load — Core automatically loads the skill on next request

Using Companion App

  1. Search for a skill (see above)
  2. Click Install on the skill
  3. Wait for download and conversion
  4. Skill is ready to use

Using Portal

  1. Search for a skill in Portal
  2. Click Install
  3. Specify version if needed (e.g., summarize@1.0.0)
  4. Wait for installation to complete

Using CLI

Install skill via CLI

python -m main skills install summarize

Install specific version

python -m main skills install summarize@1.0.0
Note: Installed skills go to external_skills/ folder. Built-in skills in skills/ are protected and cannot be removed via the API.

Skills Folders

HomeClaw loads skills from multiple directories in a specific order.

Loading Order

Skills are loaded in this order (first match wins):

  1. skills_dir — Built-in skills (skills/)
  2. external_skills_dir — ClawHub installed skills (external_skills/)
  3. skills_extra_dirs — Additional custom folders

Built-in Skills (skills/)

Protected folder containing core skills that ship with HomeClaw:

  • html-slides — Create HTML presentations
  • ppt-generation — Generate PowerPoint slides
  • image-generation — Generate images with AI
  • weather — Weather information
  • summarize — Document summarization
  • And more...

External Skills (external_skills/)

Folder for skills installed from ClawHub:

  • Skills are automatically converted to HomeClaw format
  • Can be removed via Companion app, Portal, or CLI
  • When stable, move to skills/ for permanent use

Custom Skill Folders

Add additional skill folders in core.yml:

Add custom skill folders

skills_extra_dirs:
  - /path/to/custom/skills
  - ./my-skills

Conversion Process

OpenClaw skills are automatically converted to HomeClaw format during installation.

What Gets Converted

  • SKILL.md — Preserved and adapted
  • scripts/ — Python, Shell, JavaScript files
  • references/ — Documentation and examples
  • skill.yaml — Merged into SKILL.md frontmatter

Supported Script Types

Extension Runtime Notes
.py, .pyw Python Core's Python interpreter
.sh, .bash Shell Bash shell
.js, .mjs, .cjs Node.js Requires Node.js installed
.ts TypeScript Requires tsx or ts-node

Manual Conversion

You can also manually convert an OpenClaw skill folder:

Convert skill manually

python scripts/convert_openclaw_skill.py /path/to/openclaw/skill external_skills/skill-name

Manage Skills

View, enable, disable, and remove skills through various interfaces.

List Installed Skills

List skills via CLI

python -m main skills list

Disable a Skill

Add the skill folder name to the disabled list in core.yml:

Disable skills

skills_disabled:
  - skill-name-1.0.0
  - another-skill

Remove a Skill

Only skills in external_skills/ can be removed:

Remove skill via CLI

python -m main skills remove skill-name

Or via Companion app / Portal:

  1. Go to Skills section
  2. Find the skill
  3. Click Remove
Warning: Built-in skills in skills/ cannot be removed. Use skills_disabled to disable them instead.
