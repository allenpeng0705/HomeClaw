How to Use HomeClaw

Get started with HomeClaw in three simple steps: Run Core, connect via Companion app or channels, and start chatting.

HomeClaw consists of a Core that handles all AI processing, and multiple ways to interact with it. All connections use the same Core, memory, and configuration — run Core once, use it from anywhere.

Overview

HomeClaw has three main components:

  • Core — The AI engine that processes requests, manages memory, and executes skills
  • Companion App — A dedicated desktop/mobile app for chatting with HomeClaw
  • Channels — Various platforms to interact with HomeClaw (WebChat, Telegram, etc.)

All these components share the same Core, meaning your conversations, memory, and settings are synchronized across all interfaces.

Step 1: Run Core

Start the HomeClaw Core to begin processing requests:

Start Core

python -m main start

The Core will start and listen for connections. You should see output indicating that the server is running.

Note: Keep this terminal window open while using HomeClaw. The Core needs to be running to process requests.

Configure Models

HomeClaw supports both local models (running on your machine) and cloud models (via API). You can configure them through the config/llm.yml file or using the Portal.

Suggested Models

Here are the recommended GGUF models for HomeClaw (customize in config/llm.yml):

Alternatives: Qwen3VL-2B-Instruct (lighter), Qwen3-VL-8B-Instruct (more capable), gpt-oss-20b-Q4_K_M (text-only)

Used for memory, RAG, and semantic search

Vision Model

Qwen3VL-2B-Instruct-Q4_K_M.gguf + mmproj-Qwen3VL-2B-Instruct-Q8_0.gguf

Tool Selection Model (Optional)

Fine-tuned for function calling, improves tool accuracy

Local Models

Local models run entirely on your machine, keeping your data private. HomeClaw uses llama.cpp for local model inference.

Where to Put Local Models

Place your GGUF model files in the models/ directory. Example structure:

Model directory structure

models/
├── Qwen3VL-4B-Instruct-Q4_K_M.gguf
├── mmproj-Qwen3VL-4B-Instruct-F16.gguf
├── gpt-oss-20b-Q4_K_M.gguf
└── ...

Configure Local Models in llm.yml

Edit config/llm.yml to add your local models. Here's an example from the actual configuration:

Local model configuration example

local_models:
  - id: main_vl_model_4B
    alias: main_vl_model_4B
    path: Qwen3VL-4B-Instruct-Q4_K_M.gguf
    mmproj: mmproj-Qwen3VL-4B-Instruct-F16.gguf
    host: 127.0.0.1
    port: 5026
    capabilities: [Chat]
    supported_media: [image]

  - id: gpt-oss-20b-Q4_K_M
    alias: gpt-oss-20b-Q4_K_M
    path: gpt-oss-20b-Q4_K_M.gguf
    host: 127.0.0.1
    port: 5050
    capabilities: [Chat]

Cloud Models

Cloud models provide higher performance for complex tasks. HomeClaw supports OpenAI, DeepSeek, Anthropic, Google Gemini, and other providers.

Configure Cloud Models in llm.yml

Here's an example from the actual configuration:

Cloud model configuration example

cloud_models:
  - id: DeepSeek-Chat
    alias: DeepSeek Chat
    path: deepseek/deepseek-chat
    host: 127.0.0.1
    port: 14005
    api_key_name: DEEPSEEK_API_KEY
    api_key: sk-your-api-key-here
    capabilities: [Chat]

  # OpenAI example (uncomment to use)
  # - id: OpenAI-GPT4o

  #   port: 4001
  #   api_key_name: OPENAI_API_KEY

Set API Keys

Set your API keys as environment variables (or put them directly in the api_key field):

Set API keys as environment variables

# Mac/Linux
export DEEPSEEK_API_KEY="your-key-here"
export OPENAI_API_KEY="your-key-here"

# Windows PowerShell
$env:DEEPSEEK_API_KEY="your-key-here"
$env:OPENAI_API_KEY="your-key-here"

Configure via Portal

You can also configure models through the Portal web interface:

  1. Start the Portal: python -m main portal
  2. Open http://127.0.0.1:18472 in your browser
  3. Navigate to Settings → Models
  4. Add or edit local and cloud model configurations
  5. Click Save to apply changes

Mix Mode: Best of Both Worlds

Enable Mix Mode to automatically route requests between local and cloud models based on task complexity:

Mix mode configuration in llm.yml

main_llm_mode: mix
main_llm_local: local_models/main_vl_model_4B
main_llm_cloud: cloud_models/DeepSeek-Chat

With Mix Mode enabled:

  • Simple queries use local models (free, private)
  • Complex queries use cloud models (powerful, accurate)
  • File operations (read, list) automatically route to local
  • Long outputs (after document_read, run_skill) prefer local to avoid cloud token limits

Step 2: Connect via Companion App

The Companion app provides the best experience for interacting with HomeClaw:

Download the Companion App

  • Mac: Available in clients/HomeClawApp/ directory
  • Windows: Available in clients/HomeClawApp/ directory
  • iPhone: Download from App Store (search "HomeClaw")
  • Android: Download from Google Play (search "HomeClaw")

Connect to Core

  1. Open the Companion app
  2. Enter your Core's address (default: localhost:8000)
  3. Click "Connect"
  4. Start chatting!

For detailed setup instructions, see the Companion app guide.

Step 3: Connect via Channels

HomeClaw supports multiple channels for flexibility:

WebChat

Access HomeClaw directly from your browser:

Access WebChat

http://localhost:8000/chat

Telegram

Connect HomeClaw to Telegram for on-the-go access:

  1. Create a Telegram bot via @BotFather
  2. Get your bot token
  3. Configure the Telegram channel in config/core.yml
  4. Start your bot and start chatting

Other Channels

HomeClaw also supports other channels like Slack, Discord, and more. Check the documentation for configuration details.
