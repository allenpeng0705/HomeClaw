HomeClaw 伴侣应用

与 HomeClaw 交互的最佳方式。适用于 Mac、Windows、iPhone 和 Android。

伴侣应用提供了一个美观、直观的界面，用于与 HomeClaw 聊天。它连接到您的 Core，并让您完全访问所有功能，包括记忆、插件和技能。

下载伴侣应用

伴侣应用支持所有主要平台：

桌面应用（Mac 和 Windows）

  • GitHub Releases： 从 https://github.com/allenpeng0705/HomeClaw/releases 下载
  • 从源码构建： 从仓库中的 clients/HomeClawApp/ 编译

移动应用

  • Android： 从 GitHub Releases 下载
  • iPhone： 即将上架 App Store
从源码构建： 对于所有平台，您可以从仓库中的 clients/HomeClawApp/ 构建。请参阅 GitHub 上的 HomeClawApp 和 clients/HomeClawApp/README.md 获取构建说明。

设置伴侣应用

按照以下步骤将伴侣应用连接到您的 HomeClaw Core：

步骤 1：安装应用

  • 为您的平台下载并安装伴侣应用
  • 启动应用程序

步骤 2：配置 Core URL

选择您的连接方式：

选项 A：本地局域网连接

如果您的设备与 HomeClaw Core 在同一局域网内：

  1. 找到您的 Core 的本地 IP 地址（例如 192.168.1.100）
  2. 在伴侣应用设置中输入：http://<local-ip>:9000
  3. 示例：http://192.168.1.100:9000

选项 B：通过隧道远程连接

对于 iPhone、Android 或远程访问，使用隧道服务：

  • Cloudflare Tunnel： 设置隧道以获取类似 https://your-subdomain.trycloudflare.com 的域名
  • Pinggy： 使用 Pinggy 创建带有自定义域名的隧道
  • Ngrok： 使用 Ngrok 获取公共 URL，如 https://abc123.ngrok.io

在伴侣应用中输入隧道 URL 作为您的 Core URL。

步骤 3：启用 API 密钥认证（推荐）

使用 API 密钥认证保护您的 Core 连接：

在 config/core.yml 中启用认证

auth_enable: true
auth_api_key: your-secure-api-key-here

启用认证后：

  1. 在伴侣应用中打开设置
  2. 输入您在 core.yml 中配置的相同 API 密钥
  3. 保存设置
重要： 使用远程访问（隧道）时，请始终启用 API 密钥认证。如果没有认证，任何拥有您隧道 URL 的人都可以访问您的 Core。

步骤 4：添加您的用户

通过以下两种方式之一添加您的用户：

  • 方法 1： 编辑 config/user.yml 并在 im: 下添加您的用户
  • 方法 2： 使用应用中的"管理 Core"功能（如果可用）

步骤 5：连接和聊天

  1. 在应用中点击"连接"
  2. 等待连接建立
  3. 开始与 HomeClaw 聊天！

伴侣应用功能

伴侣应用提供丰富的功能集：

美观的界面

简洁、现代的设计，带有流畅的动画和直观的导航。

完整的 Core 访问

访问所有 Core 功能，包括插件、技能和知识库。

记忆管理

查看和管理您的对话历史和记忆。

多设备同步

您的对话在连接到同一 Core 的所有设备之间同步。

富媒体支持

发送和接收图像、文件和其他媒体类型。

通知

当 HomeClaw 回复您的消息时收到通知。

离线模式

即使在离线时也可以查看过去的对话（重新连接时消息会同步）。

远程访问

使用伴侣应用从任何地方访问 HomeClaw：

二维码扫描（推荐）

使用内置的 Pinggy 隧道进行简单的远程访问：

  1. 在 config/core.yml 中设置 pinggy.token
  2. 启动 Core：python -m main start
  3. Core 启动 Pinggy 隧道并打开 http://127.0.0.1:<port>/pinggy
  4. 在伴侣应用中：设置 → 扫描二维码连接
  5. 从 /pinggy 页面扫描二维码
  6. 应用自动保存 Core URL 和 API 密钥

手动配置

使用以下方法手动配置远程访问：

  • Tailscale： 使用 Tailscale IP 作为 Core URL
  • Cloudflare Tunnel： 使用隧道 URL 作为 Core URL
  • SSH 隧道： 通过 SSH 将端口转发到 Core
安全提示： 使用远程访问时，请始终在 config/core.yml 中启用 auth_enabled 和 auth_api_key 以保护您的 Core。

故障排除

常见问题和解决方案：

无法连接到 Core

  • 确保 Core 正在运行：python -m main start
  • 检查 Core URL 是否正确（本地：http://127.0.0.1:9000 或局域网 IP）
  • 对于远程访问，验证您的隧道是否正在运行
  • 验证防火墙设置是否允许连接
  • 检查是否启用了认证以及 API 密钥是否正确

连接频繁断开

  • 检查网络稳定性
  • 尝试使用不同的远程访问方法
  • 确保 Core 在稳定的连接上运行

消息未同步

  • 检查两个设备是否都连接到同一个 Core
  • 验证 config/user.yml 中的用户配置
  • 检查 Core 日志是否有错误
