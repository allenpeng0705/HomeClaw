Cursor 和 ClaudeCode

从手机或任何设备使用 Cursor IDE 和 Claude Code CLI。

通过伴侣应用与 Cursor 和 Claude Code 聊天。打开项目、运行 AI 代理、执行命令并查看结果 — 所有这些都可以从移动设备或任何渠道完成。

概述

HomeClaw 通过运行在开发机上的桥接服务器与 Cursor IDE 和 Claude Code CLI 集成。这允许您：

  • 远程打开项目 在 Cursor IDE 中
  • 运行 AI 代理 修复 bug、编写测试、重构代码
  • 执行 shell 命令 并查看结果
  • 跨多条消息继续会话

工作原理

层 运行位置 角色
伴侣应用 手机/平板 UI，发送请求到 Core
HomeClaw Core 服务器/局域网 路由到 Cursor/ClaudeCode 插件
Cursor Bridge 开发机 运行 Cursor/Claude CLI 命令
Cursor/Claude CLI 开发机 AI 代理执行
注意： 所有文件路径指的是桥接机器的文件系统，而不是您的手机。桥接运行在安装了 Cursor 和 Claude Code 的地方。

要求

在设置 Cursor 和 ClaudeCode 好友之前，请确保在开发机上安装了必要的工具。

Cursor

  • Cursor IDE — 从 cursor.com 下载
  • Cursor CLI — 安装 shell 命令（见下文）
  • Cursor Agent — 用于运行 AI 任务的 CLI

Claude Code

  • Claude Code CLI — 从 npm 安装
  • Anthropic API 密钥 — 用于 Claude Code 认证

桥接服务器

桥接服务器随 HomeClaw 提供，运行在您的开发机上，将 Core 连接到 Cursor/Claude CLI。

安装 Cursor CLI 和 Claude Code

选项 1：使用 HomeClaw 脚本安装

最简单的方式 — 在 HomeClaw 设置期间安装两者：

Mac / Linux

安装 Cursor CLI 和 Claude Code

HOMECLAW_INSTALL_CURSOR_CLI=1 HOMECLAW_INSTALL_CLAUDE_CODE=1 bash install.sh

Windows (PowerShell)

安装 Cursor CLI 和 Claude Code

$env:HOMECLAW_INSTALL_CURSOR_CLI="1"; $env:HOMECLAW_INSTALL_CLAUDE_CODE="1"; .\install.ps1

选项 2：手动安装

Cursor CLI (Mac / Linux)

安装 Cursor CLI

curl https://cursor.com/install -fsS | bash

Cursor CLI (Windows PowerShell)

在 Windows 上安装 Cursor CLI

irm 'https://cursor.com/install?win32=true' | iex

Cursor Shell 命令

安装 cursor shell 命令以便从终端打开项目：

  1. 打开 Cursor IDE
  2. 按 Cmd+Shift+P (Mac) 或 Ctrl+Shift+P (Windows/Linux)
  3. 输入 "Shell Command: Install 'cursor' command"
  4. 重启终端

Claude Code CLI

安装 Claude Code CLI

npm install -g @anthropic-ai/claude-code

认证 Claude Code

登录 Claude Code

claude login

认证 Cursor Agent

登录 Cursor Agent

agent login

或设置环境变量：

设置 Cursor API 密钥

export CURSOR_API_KEY="your-cursor-api-key"

配置

在 config/skills_and_plugins.yml 中配置 Cursor Bridge。

自动启动桥接（推荐）

让 Core 自动启动桥接服务器：

在 skills_and_plugins.yml 中启用自动启动

cursor_bridge_auto_start: true
cursor_bridge_port: 3104

会话延续

启用会话延续，让 AI 跨消息记住上下文：

启用会话延续

cursor_bridge_cursor_continue_session: true
cursor_bridge_claude_continue_session: true

指定 CLI 路径（如需要）

如果桥接找不到 agent 或 claude，请指定完整路径：

指定 CLI 路径（Windows 示例）

cursor_bridge_agent_path: "C:\\Users\\You\\AppData\\Local\\Programs\\cursor\\agent.exe"
claude_code_path: "C:\\Users\\You\\AppData\\Local\\Programs\\claude\\claude.cmd"

API 密钥

如果认证失败，设置 API 密钥：

设置 API 密钥

cursor_bridge_cursor_api_key: "your-cursor-api-key"

完整配置示例

完整桥接配置

cursor_bridge_auto_start: true
cursor_bridge_port: 3104
cursor_bridge_forward_logs: true
cursor_bridge_cursor_continue_session: true
cursor_bridge_claude_continue_session: true
cursor_bridge_cursor_cli_path: "/usr/local/bin/cursor"
cursor_bridge_agent_path: "/usr/local/bin/agent"
claude_code_path: "/usr/local/bin/claude"

添加 Cursor/ClaudeCode 好友

将 Cursor 或 ClaudeCode 好友添加到您的用户配置中。

使用 Portal（推荐）

  1. 启动 Portal：python -m main portal
  2. 打开 http://127.0.0.1:18472
  3. 导航到用户管理
  4. 选择您的用户
  5. 点击"添加好友"
  6. 选择预设：Cursor 或 ClaudeCode
  7. 保存更改

编辑 user.yml

手动添加好友到 config/user.yml：

添加 Cursor 好友

users:
  - name: Alice
    id: alice
    username: alice
    password: your-password
    friends:
      - name: HomeClaw
      - name: Cursor
        preset: cursor

添加 ClaudeCode 好友

users:
  - name: Alice
    id: alice
    username: alice
    password: your-password
    friends:
      - name: HomeClaw
      - name: ClaudeCode
        preset: claudecode

更改后重启 Core：

重启 Core

python -m main start

从伴侣应用使用

从手机或平板与 Cursor 和 ClaudeCode 好友聊天。

打开伴侣应用

  1. 在设备上打开伴侣应用
  2. 连接到您的 HomeClaw Core
  3. 点击好友列表中的 Cursor 或 ClaudeCode

Cursor 示例命令

  • "在 Cursor 中打开 /Users/alice/myproject"
  • "运行 Cursor 代理为 auth 模块添加单元测试"
  • "运行 npm test 并显示结果"
  • "当前项目的状态是什么？"
  • "修复登录组件中的 bug"

ClaudeCode 示例命令

  • "设置项目为 /Users/alice/myapp"
  • "运行 Claude 重构数据库模块"
  • "运行 Claude 代理修复失败的测试"
  • "在项目中运行 git status"
  • "为 API 端点添加错误处理"

会话控制

伴侣应用提供快速切换：

  • Cursor： cursor_agent_yolo 切换 — 启用 --yolo 标志自动运行无需提示
  • ClaudeCode： claude_skip_permissions 切换 — 启用 --dangerously-skip-permissions

清除会话

要开始新会话：

  • "清除 cursor 会话"
  • "新 cursor 会话"
  • "清除 claude 会话"

Claude Code MCP 服务器

除了桥接（运行 Claude Code CLI），您还可以将 Claude Code 用作 MCP 服务器 — 将其内置工具直接暴露给 HomeClaw 的 LLM。

您将获得

  • HomeClaw 的 LLM 可以按名称调用 Claude Code 工具：Read、Write、Edit、Bash、Task、WebFetch 等
  • 从任何启用了 MCP 工具的好友进行细粒度控制

设置 MCP 服务器

首先，安装 MCP 客户端：

安装 MCP 客户端

pip install mcp

自动注册（推荐）

在 config/skills_and_plugins.yml 中：

自动注册 Claude Code MCP 服务器

tools:
  mcp:
    enabled: true
    auto_register_claude_code: true

手动配置

手动 MCP 服务器配置

tools:
  mcp:
    enabled: true
    servers:
      claude-code:
        transport: stdio
        command: claude
        args: [mcp, serve]
        env: {}

桥接 vs MCP

方法 最适合
桥接（ClaudeCode 好友） 自然语言任务："修复 bug"、"添加测试"。Claude Code CLI 自主运行。
MCP 服务器 细粒度工具调用：HomeClaw 的 LLM 可以按名称调用 Read、Write、Edit、Bash。

您可以同时使用两者：ClaudeCode 好友用于交互式 CLI 任务，MCP 用于程序化工具访问。

提示

桥接必须运行

桥接服务器必须在您的开发机上运行，Cursor/ClaudeCode 好友才能工作。启用 cursor_bridge_auto_start: true 后，Core 会自动启动它。

检查桥接状态

检查桥接健康状态

curl http://127.0.0.1:3104/health

远程访问

如果 Core 运行在与桥接不同的机器上：

  1. 在插件配置中将 base_url 设置为桥接机器的 IP
  2. 在桥接机器的防火墙上允许入站 TCP 3104
  3. 使用 Tailscale 或 VPN 进行安全访问

会话延续

启用会话延续后，AI 会跨消息记住上下文。每个项目都有自己的会话，因此切换项目会维护独立的上下文。

查看日志

启用日志转发以在 Core 终端中查看桥接输出：

启用桥接日志

cursor_bridge_forward_logs: true

安全

警告： 桥接在您的开发机上运行，可以执行任意 shell 命令。仅在受信任的环境中运行。如果暴露桥接端口，请考虑添加 API 密钥认证。
