HomeClaw Portal

本地 Web 界面，用于管理和配置 HomeClaw。

Portal 提供便捷的 Web 界面来配置您的 HomeClaw 实例。使用它来管理模型、配置设置和监控系统 — 全部通过浏览器完成。

访问 Portal

Portal 在本地运行，可通过浏览器访问：

启动 Portal

启动 Portal

python -m main portal

访问 Portal

启动后，在浏览器中打开：

Portal URL

http://127.0.0.1:18472
注意： Portal 仅供本地访问。它提供了便捷的界面来管理 HomeClaw 配置，无需直接编辑 YAML 文件。

Portal 功能

Portal 提供全面的管理功能：

模型配置

通过直观的界面配置本地和云端模型：

  • 添加和配置本地模型（GGUF 文件）
  • 设置云端模型提供商（OpenAI、DeepSeek 等）
  • 配置混合模式以获得最佳性能
  • 设置主 LLM 和嵌入模型的默认模型

设置管理

从 Portal 管理所有 HomeClaw 设置：

  • 编辑 core.yml 设置
  • 配置用户偏好
  • 管理 LLM 参数
  • 设置认证

插件管理

直接从 Portal 管理插件：

  • 查看已安装的插件
  • 启用或禁用插件
  • 配置插件设置
  • 监控插件状态

频道配置

配置通信频道：

  • 设置 Telegram 机器人
  • 配置 Discord 集成
  • 管理 WebChat 设置
  • 测试频道连接

系统监控

监控您的 HomeClaw 实例：

  • 查看系统状态
  • 检查模型可用性
  • 监控资源使用
  • 查看最近日志

管理工具

Portal 提供强大的管理功能：

配置文件

通过 Web 界面编辑配置文件：

  • core.yml — Core 主配置
  • llm.yml — 模型和 LLM 设置
  • user.yml — 用户档案和偏好
  • plugins/ — 插件配置

服务控制

从 Portal 控制 HomeClaw 服务：

  • 启动和停止 Core
  • 重启服务
  • 检查服务状态
  • 查看服务日志

数据管理

管理您的 HomeClaw 数据：

  • 查看对话历史
  • 导出和导入配置
  • 备份设置
  • 清除缓存数据

认证设置

配置安全设置：

  • 启用/禁用 API 密钥认证
  • 设置或更改 API 密钥
  • 配置允许的主机
  • 管理访问权限

在 core.yml 中启用认证

auth_enable: true
auth_api_key: your-secure-api-key-here

故障排除

常见的 Portal 问题和解决方案：

无法访问 Portal

  • 确保 Portal 正在运行：python -m main portal
  • 检查端口 18472 是否可用
  • 验证 URL 是否正确：http://127.0.0.1:18472
  • 检查是否有其他进程占用端口

Portal 无法启动

  • 检查 Python 版本（需要 3.11+）
  • 验证所有依赖已安装
  • 检查 Portal 日志中的错误
  • 尝试使用不同端口：python -m main portal --port 18473

配置更改未保存

  • 检查 config/ 目录的文件权限
  • 验证 YAML 语法是否正确
  • 检查是否有冲突的设置
  • 更改后重启 Portal
