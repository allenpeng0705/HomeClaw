远程访问

从世界任何地方连接到您的 HomeClaw。

使用安全的隧道解决方案访问您的家庭社交网络、与 AI 好友聊天，并远程管理您的 HomeClaw。完美支持 iPhone、Android 和远程桌面访问。

概述

当您离开家时，您仍然可以使用各种隧道解决方案访问您的 HomeClaw 和家庭社交网络。这些方案在您的设备和 HomeClaw 服务器之间创建安全连接。

为什么需要远程访问？

  • iPhone/Android： 在任何地方使用移动设备上的伴侣应用
  • 家庭社交网络： 远程与家人保持联系
  • AI 好友： 随时随地与您的 AI 助手聊天
  • 远程工作： 旅行或工作时访问 HomeClaw

连接方式

  • 本地局域网： 在同一网络上时使用本地 IP（例如 http://192.168.1.100:9000）
  • 隧道服务： 离家时使用（Pinggy、Cloudflare、Ngrok 等）
重要： 使用远程访问时，请始终启用 API 密钥认证（auth_enable: true 和 auth_api_key）。这可以防止未经授权访问您的 HomeClaw。

Pinggy（内置）

Pinggy 是设置远程访问的最简单方式。HomeClaw 内置支持 Pinggy，支持二维码扫描。

设置步骤

  1. 从 pinggy.io 获取 Pinggy 令牌
  2. 将令牌添加到您的 config/core.yml：

在 core.yml 中配置 Pinggy

pinggy:
  token: your-pinggy-token-here
  enabled: true
  1. 启动 HomeClaw Core：python -m main start
  2. Core 将自动启动 Pinggy 隧道
  3. 在浏览器中打开 Pinggy URL（例如 http://127.0.0.1:9000/pinggy）
  4. 使用伴侣应用扫描二维码

二维码连接

连接伴侣应用的最简单方式：

  1. 在您的 HomeClaw 服务器上，打开 http://127.0.0.1:9000/pinggy
  2. 将显示一个二维码
  3. 在伴侣应用中：设置 → 扫描二维码连接
  4. 扫描二维码
  5. 应用自动配置 URL 和 API 密钥

优势

  • 内置于 HomeClaw - 无需额外软件
  • 二维码扫描，方便移动设备设置
  • 可使用自定义域名
  • 安全加密连接

Cloudflare Tunnel

Cloudflare Tunnel 提供免费、安全的隧道，带有自定义子域名。基本使用无需账户。

设置步骤

  1. 在您的 HomeClaw 服务器上安装 cloudflared：

安装 cloudflared（Mac/Linux）

brew install cloudflared

安装 cloudflared（Windows）

winget install --id Cloudflare.cloudflared
  1. 创建隧道：

创建 Cloudflare 隧道

cloudflared tunnel --url http://localhost:9000
  1. 您将获得类似 https://your-subdomain.trycloudflare.com 的 URL
  2. 在伴侣应用中使用此 URL 作为 Core URL

持久隧道（可选）

使用自定义子域名的永久隧道：

  1. 创建 Cloudflare 账户
  2. 认证 cloudflared：cloudflared login
  3. 创建命名隧道：cloudflared tunnel create homeclaw
  4. 在 Cloudflare 仪表板中配置隧道
  5. 作为服务运行以自动启动

优势

  • 免费套餐，额度 generous
  • 快速隧道无需账户
  • 账户可使用自定义子域名
  • 高度可靠和快速

Ngrok

Ngrok 是一个流行的隧道服务，为您的本地服务器提供即时公共 URL。

设置步骤

  1. 在 ngrok.com 注册
  2. 下载并安装 ngrok
  3. 使用您的令牌认证：

认证 ngrok

ngrok authtoken your-token-here
  1. 启动隧道：

启动 ngrok 隧道

ngrok http 9000
  1. 您将获得类似 https://abc123.ngrok.io 的 URL
  2. 在伴侣应用中使用此 URL

自定义域名（付费）

使用付费 ngrok 套餐，您可以使用自定义域名：

自定义域名隧道

ngrok http --domain=your-domain.ngrok.io 9000

优势

  • 易于设置和使用
  • 提供免费套餐
  • Web 界面用于检查流量
  • 付费套餐可使用自定义域名

Tailscale

Tailscale 在您的设备之间创建安全的网状 VPN 网络。它就像您在同一个本地网络上一样工作，即使远程也是如此。

设置步骤

  1. 在 tailscale.com 注册
  2. 在您的 HomeClaw 服务器上安装 Tailscale：

安装 Tailscale

# Mac
brew install tailscale

# Linux
curl -fsSL https://tailscale.com/install.sh | sh

# Windows
# 从 tailscale.com 下载
  1. 启动 Tailscale 并认证：

启动 Tailscale

sudo tailscale up
  1. 获取您的 HomeClaw 服务器的 Tailscale IP：

检查 Tailscale IP

tailscale ip -4
  1. 在您的移动设备或远程计算机上安装 Tailscale
  2. 连接到同一个 Tailscale 网络
  3. 在伴侣应用中使用 Tailscale IP：http://100.x.x.x:9000

优势

  • 像本地网络一样工作 - 无需公共 URL
  • 个人使用免费（最多 20 台设备）
  • 非常安全 - 使用 WireGuard 加密
  • 无需端口转发
  • 跨多台设备无缝工作

SSH 隧道

如果您有一台具有公共 IP 的服务器，您可以使用 SSH 隧道将连接转发到您的 HomeClaw。

设置步骤

  1. 拥有一台具有 SSH 访问权限和公共 IP 的服务器
  2. 从您的 HomeClaw 服务器，创建 SSH 隧道：

创建 SSH 隧道

ssh -R 9000:localhost:9000 user@your-server.com
  1. 这将从您的服务器转发端口 9000 到 HomeClaw
  2. 在伴侣应用中使用您服务器的 IP：http://your-server.com:9000

持久 SSH 隧道

使用 autossh 实现持久隧道：

安装并使用 autossh

# 安装 autossh
brew install autossh  # Mac
sudo apt install autossh  # Linux

# 创建持久隧道
autossh -M 0 -N -R 9000:localhost:9000 user@your-server.com

优势

  • 如果您有服务器，则不需要额外的服务
  • 非常安全（SSH 加密）
  • 完全控制连接
  • 不需要第三方服务

安全

将 HomeClaw 暴露到互联网时，安全至关重要。

启用认证

始终在 config/core.yml 中启用 API 密钥认证：

启用认证

auth_enable: true
auth_api_key: your-very-secure-random-key-here

安全最佳实践

  • 使用强 API 密钥： 生成长且随机的 API 密钥
  • 定期轮换密钥： 定期更改您的 API 密钥
  • 仅使用 HTTPS： 尽可能使用 HTTPS 隧道（Cloudflare、Ngrok HTTPS）
  • 限制访问： 仅与受信任的家庭成员共享隧道 URL
  • 监控连接： 检查 Core 日志以发现未经授权的访问尝试

防火墙注意事项

  • 隧道服务处理外部访问 - 不需要端口转发
  • 对于 Tailscale，确保防火墙允许 Tailscale 流量
  • SSH 隧道需要 SSH 端口（22）可访问
警告： 切勿在未启用认证的情况下将 HomeClaw 暴露到互联网。如果没有 auth_enable: true，任何拥有您隧道 URL 的人都可以访问您的 Core 和数据。

连接伴侣应用

设置隧道后，连接您的伴侣应用以远程访问 HomeClaw。

连接步骤

  1. 在您的设备上打开伴侣应用
  2. 进入 设置
  3. 输入隧道 URL 作为 Core URL：
  • Pinggy：https://xxx.pinggy.io
  • Cloudflare：https://your-name.trycloudflare.com
  • Ngrok：https://abc123.ngrok.io
  • Tailscale：http://100.x.x.x:9000
  • SSH：http://your-server.com:9000
  1. 输入 config/core.yml 中的 API 密钥
  2. 保存设置
  3. 点击 连接
