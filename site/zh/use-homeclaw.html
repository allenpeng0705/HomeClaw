如何使用 HomeClaw

通过三个简单步骤开始使用 HomeClaw：运行 Core，通过伴侣应用或渠道连接，然后开始聊天。

HomeClaw 由处理所有 AI 请求的 Core 和多种交互方式组成。所有连接使用相同的 Core、内存和配置 — 运行 Core 一次，随处使用。

概述

HomeClaw 有三个主要组件：

  • Core — 处理请求、管理内存和执行技能的 AI 引擎
  • 伴侣应用 — 与 HomeClaw 聊天的专用桌面/移动应用
  • 渠道 — 与 HomeClaw 交互的各种平台（WebChat、Telegram 等）

所有这些组件共享相同的 Core，意味着您的对话、记忆和设置在所有界面之间同步。

步骤 1：运行 Core

启动 HomeClaw Core 以开始处理请求：

启动 Core

python -m main start

Core 将启动并监听连接。您应该看到输出指示服务器正在运行。

注意：在使用 HomeClaw 时保持此终端窗口打开。Core 需要运行才能处理请求。

配置模型

HomeClaw 支持本地模型（在您的机器上运行）和云端模型（通过 API）。您可以通过 config/llm.yml 文件或使用 Portal 来配置它们。

推荐模型

以下是 HomeClaw 推荐的 GGUF 模型（可在 config/llm.yml 中自定义）。只需主聊天模型即可 — 其他为可选增强功能：

必需：主聊天模型 — 这是您开始使用所需的唯一模型。

主聊天模型

Qwen3VL-4B-Instruct-Q4_K_M.gguf + mmproj-Qwen3VL-4B-Instruct-F16.gguf

备选：Qwen3VL-2B-Instruct（更轻量）、Qwen3-VL-8B-Instruct（更强大）、gpt-oss-20b-Q4_K_M（纯文本）

嵌入模型

Qwen3-Embedding-0.6B-Q8_0.gguf

用于记忆、RAG 和语义搜索

视觉模型

Qwen3VL-2B-Instruct-Q4_K_M.gguf + mmproj-Qwen3VL-2B-Instruct-Q8_0.gguf

用于图像理解（当主模型为纯文本时）

工具选择模型（可选）

Qwen3-4B-Function-Calling-Pro.gguf

专用于函数调用，提高工具准确性

本地模型

本地模型完全在您的机器上运行，保持数据私密。HomeClaw 使用 llama.cpp 进行本地模型推理。

本地模型存放位置

将 GGUF 模型文件放在 models/ 目录中。示例结构：

模型目录结构

models/
├── Qwen3VL-4B-Instruct-Q4_K_M.gguf
├── mmproj-Qwen3VL-4B-Instruct-F16.gguf
├── gpt-oss-20b-Q4_K_M.gguf
└── ...

在 llm.yml 中配置本地模型

编辑 config/llm.yml 以添加您的本地模型。以下是实际配置示例：

本地模型配置示例

local_models:
  - id: main_vl_model_4B
    alias: main_vl_model_4B
    path: Qwen3VL-4B-Instruct-Q4_K_M.gguf
    mmproj: mmproj-Qwen3VL-4B-Instruct-F16.gguf
    host: 127.0.0.1
    port: 5026
    capabilities: [Chat]
    supported_media: [image]

  - id: gpt-oss-20b-Q4_K_M
    alias: gpt-oss-20b-Q4_K_M
    path: gpt-oss-20b-Q4_K_M.gguf
    host: 127.0.0.1
    port: 5050
    capabilities: [Chat]

下载模型

您可以从以下位置下载 GGUF 格式的模型：

  • Hugging Face — 搜索 "gguf" 模型
  • Ollama 库 — 将 Ollama 模型转换为 GGUF

云端模型

云端模型为复杂任务提供更高的性能。HomeClaw 支持 OpenAI、DeepSeek、Anthropic、Google Gemini 和其他提供商。

在 llm.yml 中配置云端模型

以下是实际配置示例：

云端模型配置示例

cloud_models:
  - id: DeepSeek-Chat
    alias: DeepSeek Chat
    path: deepseek/deepseek-chat
    host: 127.0.0.1
    port: 14005
    api_key_name: DEEPSEEK_API_KEY
    api_key: sk-your-api-key-here
    capabilities: [Chat]

  # OpenAI 示例（取消注释以使用）
  # - id: OpenAI-GPT4o
  #   alias: OpenAI GPT-4o
  #   path: openai/gpt-4o
  #   host: 127.0.0.1
  #   port: 4001
  #   api_key_name: OPENAI_API_KEY
  #   supported_media: [image]
  #   capabilities: [Functioncall, Chat]

设置 API 密钥

将您的 API 密钥设置为环境变量（或直接放在 api_key 字段中）：

设置 API 密钥为环境变量

# Mac/Linux
export DEEPSEEK_API_KEY="your-key-here"
export OPENAI_API_KEY="your-key-here"

# Windows PowerShell
$env:DEEPSEEK_API_KEY="your-key-here"
$env:OPENAI_API_KEY="your-key-here"

通过 Portal 配置

您还可以通过 Portal Web 界面配置模型：

  1. 启动 Portal：python -m main portal
  2. 在浏览器中打开 http://127.0.0.1:18472
  3. 导航到 设置 → 模型
  4. 添加或编辑本地和云端模型配置
  5. 点击 保存 应用更改

混合模式：两全其美

启用混合模式以根据任务复杂度自动在本地和云端模型之间路由请求：

llm.yml 中的混合模式配置

main_llm_mode: mix
main_llm_local: local_models/main_vl_model_4B
main_llm_cloud: cloud_models/DeepSeek-Chat

启用混合模式后：

  • 简单查询使用本地模型（免费、私密）
  • 复杂查询使用云端模型（强大、准确）
  • 文件操作（读取、列表）自动路由到本地
  • 长输出（在 document_read、run_skill 之后）优先使用本地以避免云端令牌限制

步骤 2：通过伴侣应用连接

伴侣应用提供了与 HomeClaw 交互的最佳体验：

下载伴侣应用

  • Mac： 可在 clients/HomeClawApp/ 目录中找到
  • Windows： 可在 clients/HomeClawApp/ 目录中找到
  • iPhone： 从 App Store 下载（搜索 "HomeClaw"）
  • Android： 从 Google Play 下载（搜索 "HomeClaw"）

连接到 Core

  1. 打开伴侣应用
  2. 输入您的 Core 地址（默认：localhost:8000）
  3. 点击"连接"
  4. 开始聊天！

有关详细设置说明，请参阅 伴侣应用指南。

步骤 3：通过渠道连接

HomeClaw 支持多种渠道以提供灵活性：

WebChat

直接从浏览器访问 HomeClaw：

访问 WebChat

http://localhost:8000/chat

Telegram

将 HomeClaw 连接到 Telegram 以实现随时随地访问：

  1. 通过 @BotFather 创建 Telegram 机器人
  2. 获取您的机器人令牌
  3. 在 config/core.yml 中配置 Telegram 渠道
  4. 启动您的机器人并开始聊天

其他渠道

HomeClaw 还支持其他渠道，如 Slack、Discord 等。查看 文档 了解配置详情。

连接后，您可以利用 HomeClaw 的强大功能：

混合模式

同时使用云端和本地 AI 模型以获得最佳性能和成本效益。

插件和技能

使用任何语言编写的插件和技能扩展 HomeClaw 的功能。

记忆

HomeClaw 在会话之间记住您的对话和上下文。

知识库

上传文档，让 HomeClaw 从您的个人知识库中学习。

多智能体

创建多个专门在复杂任务上协同工作的智能体。

最佳体验提示

  • 保持 Core 运行： Core 必须运行才能处理请求
  • 使用伴侣应用： 伴侣应用提供最佳用户体验
  • 配置模型： 在 config/core.yml 中设置您偏好的 AI 模型
  • 启用混合模式： 同时使用云端和本地模型以获得最佳性能
  • 探索插件： 安装插件以扩展功能
  • 检查日志： 监控 Core 日志以进行故障排除
